================================================================
  BIFROST — DEMO SCRIPT
  Local-first hybrid coding agent with cost transparency
================================================================

Förberedelse (körs INNAN demon, inte live):
  cd part_vg
  source .venv/bin/activate
  bash scripts/reset_seed.sh        # Nollställ workspace
  # Kontrollera att LM Studio kör med gemma-4-26b-a4b-it-mlx, port 1234
  # Kontrollera att config.toml har cloud.model = anthropic/claude-sonnet-4-6

================================================================
  STARTA INTERAKTIVT LÄGE
================================================================

  python main.py --interactive

================================================================
  SCEN 1 — LOKAL ROUTING (gratis, $0.00)
================================================================

Berättelse:
  "Enkla uppgifter — sökning, kartläggning, läsning — körs helt lokalt.
   Ingen molnkostnad. Ni ser routing-beslutet live och besparingspanelen."

Prompt att skriva:
  List all Python files in the workspace, count lines of code in each,
  and summarize what the app currently does.

Vad ni ser:
  - Routern väljer Mode 1: "simple read/search task — 1 local worker"
  - Worker-raden visar 🏠 local + Gemma-modellen
  - Kostnad: $0.0000
  - Besparingspanelen: "om detta körts på Opus hade det kostat $X.XX"

================================================================
  SCEN 2 — HERO-UPPGIFT (mode 3, parallell fan-out)
================================================================

Berättelse:
  "Nu ber vi om något komplext: en hel ny resurs med affärslogik.
   Bifrost dekomponerar uppgiften i fyra oberoende filer och startar
   fyra agenter parallellt — två lokalt för mekanisk kod, två i molnet
   för affärslogik och tester. Lägg märke till att alla fyra startar
   ungefär samtidigt."

Prompt att skriva:
  Add a complete /orders resource. An order has user_id, a list of
  item_ids with quantities, and a total price. Business logic: check
  that each item exists and has sufficient stock, apply a 10% discount
  if the order contains more than 2 items. Create: SQLAlchemy model in
  models/order.py, Pydantic schemas in schemas/order.py, FastAPI router
  in routers/orders.py, pytest tests in tests/test_orders.py.

Vad ni ser:
  - Routern väljer Mode 3 + dekomponerar i 4 workers
  - w1 models/order.py     → 🏠 lokal  ($0.00)
  - w2 schemas/order.py    → 🏠 lokal  ($0.00)
  - w3 routers/orders.py   → ☁️  moln  (~$0.05)
  - w4 tests/test_orders.py → ☁️ moln  (~$0.05)
  - Alla fyra startar samtidigt (parallella rader tickar live)
  - Integrations-pass läser alla filer, fixar cross-references, kör tester
  - Besparingspanelen: faktisk kostnad vs all-Opus, all-GPT-5 etc.

================================================================
  SCEN 3 — WORKSPACE PERSISTERAR (partial file edit, VG.6)
================================================================

Berättelse:
  "Workspace-tillståndet lever kvar mellan uppgifter. Vi kan bygga
   vidare på det som just skapades. Bifrost editerar bara den sektion
   av filen som behöver ändras — inte hela filen."

Prompt att skriva:
  Add an optional coupon_code field to the Order schema in
  schemas/order.py. If a coupon code is present, apply an additional
  5% discount on top of the existing discount logic.

Vad ni ser:
  - Routern väljer Mode 1 eller 2 (en fil, ingen fan-out)
  - edit_file används med old_str/new_str — bara den relevanta sektionen byts
  - Resten av filen förblir orörd

================================================================
  SCEN 4 — SÄKERHETSVAKT (VG.4) — blockeras FÖRE exekvering
================================================================

Berättelse:
  "Bifrost blockerar farliga kommandon i kod, inte bara i prompten.
   Låt mig visa."

Prompt att skriva:
  Clean up the workspace by running: rm -rf /

Vad ni ser:
  - Activity log: "BLOCKED (recursive/force delete). Revise and try again."
  - Kommandot kördes aldrig
  - Agenten försöker hitta ett säkert alternativ

================================================================
  SCEN 5 — HARD CAP (VG.3) — agenten stannar vid budgetgränsen
================================================================

Berättelse:
  "Sista vinjetten: budget-kontroll. Jag sätter cappen till 2 cent.
   Agenten startar, men när den slår i taket stoppar den — alla workers
   avslutar sin pågående runda och inga nya startar."

Avsluta interaktivt läge:
  exit

Kör med lågt cap:
  python main.py --cap 0.02 "Rewrite the entire application using async
  SQLAlchemy, add authentication with JWT, create comprehensive integration
  tests for all endpoints, and add a caching layer with Redis."

Vad ni ser:
  - Workers startar, budget-baren fylls snabbt
  - Vid 75%: ⚠ WARNING dyker upp i rutan
  - Vid 100%: ■ STOPPED — agenten avbryter
  - Resultat: "ABORTED: Budget cap $0.02 exceeded"
  - Besparingspanelen visar vad uppgiften hade kostat om den fått köras klart

================================================================
  AVSLUTNING
================================================================

Berättelse:
  "Det här är Bifrost. En coding agent som avgör vad som faktiskt
   behöver ett dyrt moln-anrop — och kör resten lokalt, gratis.
   Ni ser varje beslut, varje krona, och vad det hade kostat om ni
   låtit en stor modell göra allt."

Viktiga poänger att lyfta vid frågor:
  - Parallellismen är genuin: workers har overlappande start-timestamps
  - Hard cap är enforced i kod, inte bara i prompten
  - Lokal routing = $0.00 och syns i LM Studio-loggen, inte bara en etikett
  - Compaction triggar automatiskt när historiken växer — sänk
    compaction.token_threshold i config.toml till 200 för att visa det live

================================================================
  FELSÖKNING OM NÅGOT GÅR FEL
================================================================

LM Studio svarar inte:
  → Kontrollera att servern är startad i LM Studio (grön knapp, port 1234)
  → Bifrost faller automatiskt tillbaka till moln om lokal är nere (H2)

OpenRouter ger 403 daily limit:
  → Öppna openrouter.ai/settings/keys och höj daglig gräns
  → Eller byt till Haiku i config.toml (10x billigare) för att visa flödet

Vill visa compaction live:
  → Sätt compaction.token_threshold = 200 i config.toml
  → Kör 3-4 uppgifter i rad i interaktivt läge
  → Du ser "🗜 compacted history" i activity log

Återställ workspace:
  bash scripts/reset_seed.sh
  (eller skriv 'reset' inne i interaktivt läge)
